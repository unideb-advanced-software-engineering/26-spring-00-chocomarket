Welcome to the documentation of **ChocoMarket**, a case study developed as part of the **Zamunda Digital Renaissance (ZDR)** program.

---

## 🌎 Overview

ChocoMarket is a nationwide platform designed to aggregate and present chocolate manufacturers’ inventory data in a consistent, accessible way.

It addresses key challenges such as:

- Fragmented data sources (FTP, email, APIs, etc.)
- Multiple file formats (XML, CSV, JSON)
- Limited comparability of products and stock

---

## ⚙️ Purpose of This Documentation

This documentation provides a clear and structured overview of:

- System requirements  
- Core functionalities  
- Design decisions  
- Technical constraints  

---

## 👁 Design Focus

The system is built with emphasis on:

- **Reliability** in handling diverse data inputs  
- **Cost-efficiency** in line with government program goals  
- **Adaptability** to real-world infrastructure limitations  
  (e.g., variable network quality and bandwidth)

---

## ➤ Goal

ChocoMarket aims to support manufacturers and interested partners by offering a unified, transparent view of chocolate inventories—while laying the foundation for future expansion.
